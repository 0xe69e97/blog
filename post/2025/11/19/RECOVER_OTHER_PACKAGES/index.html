Monorepo 变动后，恢复其他 Package 中被误修改的文件

问题场景

在 monorepo 项目中，当你在某个 package（如 currentProject）中开发功能时，可能不小心修改了其他 package（如 otherProject1、otherProject2）的文件。如果这些修改已经提交到了 git，需要恢复到之前的版本。

解决步骤

1. 查看最近的提交记录

首先查看最近的提交，确认哪些文件被修改了：

# 查看最近一次提交修改的文件
git show --name-status HEAD

# 或者查看提交统计信息
git show HEAD --stat

输出示例：

M       packages/otherProject1/src/components/Form/form-item.vue
M       packages/otherProject1/src/components/item-picker/index.vue
M       packages/currentProject/src/api/index.js
M       packages/otherProject2/src/components/item-picker/index.vue

2. 确认需要恢复的文件

从提交记录中，识别出不属于当前功能开发范围的其他 package 文件。

例如：

  • ✅ 需要恢复：packages/otherProject1/... 和 packages/otherProject2/... 中的文件
  • ❌ 不需要恢复：packages/currentProject/... 中的文件（这是当前开发的功能）

3. 恢复文件到上一个版本

使用 git checkout 命令将文件恢复到上一个提交版本（HEAD~1）：

git checkout HEAD~1 -- <文件路径1> <文件路径2> ...

实际示例：

git checkout HEAD~1 -- \
  packages/otherProject1/src/components/Form/form-item.vue \
  packages/otherProject1/src/components/item-picker/index.vue \
  packages/otherProject2/src/components/item-picker/index.vue \
  packages/otherProject2/src/mixins/commonMethods.js

说明：

  • HEAD~1 表示上一个提交版本
  • -- 后面跟要恢复的文件路径
  • 可以一次恢复多个文件，用空格分隔

4. 检查恢复结果

查看 git 状态，确认文件已恢复到暂存区：

git status

输出示例：

Changes to be committed:
  (use "git restore --staged <file>..." to unstage)
        modified:   packages/otherProject1/src/components/Form/form-item.vue
        modified:   packages/otherProject1/src/components/item-picker/index.vue
        modified:   packages/otherProject2/src/components/item-picker/index.vue
        modified:   packages/otherProject2/src/mixins/commonMethods.js

5. 查看恢复的差异（可选）

如果需要确认恢复的内容是否正确，可以查看差异：

# 查看暂存区的差异统计
git diff --cached --stat

# 查看详细的差异内容
git diff --cached

6. 提交恢复的文件

确认无误后，提交这些恢复的文件：

git commit -m "revert: 恢复 otherProject1 和 otherProject2 中被误修改的文件"

其他恢复方式

方式一：恢复到指定提交版本

如果需要恢复到更早的版本，可以指定具体的提交 hash：

git checkout <commit-hash> -- <文件路径>

例如：

git checkout d0428a75 -- packages/otherProject1/src/components/Form/form-item.vue

方式二：恢复到远程分支版本

如果需要恢复到远程分支的版本：

git checkout origin/master -- <文件路径>

方式三：使用 git restore（Git 2.23+）

新版本的 Git 推荐使用 git restore 命令：

# 恢复工作区文件
git restore --source=HEAD~1 -- <文件路径>

# 恢复并暂存
git restore --source=HEAD~1 --staged -- <文件路径>

注意事项

  1. 确认文件列表：恢复前务必确认哪些文件需要恢复，避免误操作
  2. 备份重要修改：如果文件中有重要修改，建议先备份
  3. 检查依赖关系：恢复文件后，检查是否有其他文件依赖这些修改
  4. 测试验证：恢复后建议运行测试，确保功能正常

完整操作流程示例

# 1. 查看最近提交的文件
git show --name-status HEAD

# 2. 恢复其他 package 的文件
git checkout HEAD~1 -- \
  packages/otherProject1/src/components/Form/form-item.vue \
  packages/otherProject1/src/components/item-picker/index.vue \
  packages/otherProject2/src/components/item-picker/index.vue \
  packages/otherProject2/src/mixins/commonMethods.js

# 3. 检查状态
git status

# 4. 查看差异（可选）
git diff --cached --stat

# 5. 提交恢复的文件
git commit -m "revert: 恢复其他 package 中被误修改的文件"

常见问题

Q: 如果文件已经 push 到远程了怎么办？

A: 恢复文件后，需要 push 新的提交：

git commit -m "revert: 恢复误修改的文件"
git push

Q: 如何恢复整个 package？

A: 可以恢复整个目录：

git checkout HEAD~1 -- packages/otherProject1/

Q: 恢复后想撤销怎么办？

A: 如果还没有提交，可以使用：

git restore --staged <文件路径>
git restore <文件路径>
